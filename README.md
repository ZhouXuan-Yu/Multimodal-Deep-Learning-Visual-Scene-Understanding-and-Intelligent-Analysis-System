
## 📋 项目概述

本项目构建了一个**基于多模态深度学习的视觉场景理解与智能分析系统**，系统在人物识别任务中构建了 YOLO + ResNet50 + ResNet18 + DeepLabV3 + Qwen2.5 的多模型级联架构，实现从目标检测、视觉特征提取到语义分割与高层语义理解的多阶段融合分析，有效提升复杂场景下的人物识别准确率。同时，将视觉识别结果与大语言模型（Qwen2.5）结合，构建图像—文本多模态理解框架，实现基于自然语言的图像分析对话与智能路径规划功能，并通过语义解析将自然语言需求转化为结构化路径参数，结合高德地图 API 实现多策略路径规划。此外，系统集成 AI 邮件自动化处理模块，可对用户发送的信息进行智能分析与结构化整理，并自动发送至指定邮箱，形成完整的智能安防信息处理流程。

### 🎯 四大核心功能

1. **智程导航** - 基于LLM的智能路径规划
   - 自然语言路径规划指令解析（Qwen2.5）
   - 多策略路径规划（最短路径、最快路径、避堵路径）
   - 高德地图API集成与可视化

2. **智眸千析** - 多模态图像识别与分析
   - YOLO系列目标检测（人物、车牌等）
   - 人物属性识别（年龄、性别、服装颜色）
   - 图像-文本多模态融合与LLM智能对话

3. **智航监控** - 车牌监控与智能告警
   - 车牌检测与识别（YOLO + CNN+LSTM）
   - 视频分析与状态向量建模
   - 目标匹配与智能告警机制

4. **邮件自动化** - 智能告警通知系统
   - 基于检测结果的智能决策
   - 邮件通知系统（告警摘要、关键帧截图）
   - 数据驱动决策流程

## 🔬 核心技术亮点

### 1. 多模型融合架构
- **YOLO + ResNet + 颜色识别**的级联融合架构
- 通过**置信度加权策略**融合多模型输出，提升识别准确率
- 车牌识别流程：YOLO检测 → 裁剪 → CNN+LSTM识别 → 颜色识别 → 多模型结果融合
- 人物识别流程：YOLO检测 → 裁剪 → ResNet特征提取 → 年龄/性别/颜色识别 → 多模型结果融合

### 2. 视觉-文本多模态融合
- **图像识别结果与LLM语义理解的深度融合**
- 图像分析上下文构建：将YOLO检测结果转换为结构化文本描述
- 系统提示词设计：构建包含图像分析结果的系统提示词
- 流式对话：使用流式API实现与LLM的实时对话
- 多轮对话管理：维护对话历史，支持基于图像分析结果的多轮问答

### 3. 基于LLM的路径规划语义解析
- **自然语言指令解析**：将自然语言指令转换为API调用参数
- ROUTE_PROMPT设计：设计面向路径规划场景的Prompt模板
- 结构化参数提取：从自然语言描述中提取起点、终点、途经点、出行偏好等
- 多策略路径规划：最短路径、最快路径、避堵路径等
- 多目标优化：将规划问题建模为多目标优化问题，通过加权综合评价函数选择最优路线

### 4. 状态向量建模
- **多源状态向量建模**：车牌号、位置、时间戳、置信度、车辆颜色等多维状态信息
- 视频场景抽象：将视频帧序列抽象为状态向量序列
- 状态更新机制：通过目标追踪和检测结果更新状态向量
- 状态匹配：支持精确匹配、子串匹配、相似度匹配等多种匹配策略

## 🚀 快速开始

### 环境要求

- **操作系统**：Windows 10/11（当前开发环境为 Windows 10）
- **Python**：3.8 及以上版本（推荐 3.10）
- **Node.js**：16 及以上版本
- **CUDA**：建议 12.4 及以上（如需启用 GPU 推理）
- **包管理工具**：
  - 后端：`conda`（推荐）或 `venv`
  - 前端：`npm` / `pnpm`
- **建议的 Conda 环境名称**：`modelapp`

### 安装步骤

#### 1. 克隆仓库

```bash
git clone https://github.com/ZhouXuan-Yu/BiShe.git
cd BiShe/ModelService_graduation-main
```

#### 2. 后端环境配置

```bash
# 创建并激活 conda 环境
conda create -n modelapp python=3.10
conda activate modelapp

# 安装后端依赖
cd ModelService_graduation-main
pip install -r requirements.txt  # 如有 requirements.txt
# 或根据各子模块的 requirements.txt 分别安装
```

#### 3. 启动后端服务

**方式一：使用统一启动脚本（推荐）**

```bash
cd D:\ModelService_graduation-main\ModelService_graduation-main
conda activate modelapp
python start_service.py --host 0.0.0.0 --port 8082
```

**方式二：直接使用 Uvicorn**

```bash
cd ModelService_graduation-main/ModelService/Main
conda activate modelapp
python -m uvicorn app.main:app --host 0.0.0.0 --port 8082 --reload
```

启动成功后，访问：
- **Swagger API 文档**：`http://localhost:8082/docs`
- **ReDoc 文档**：`http://localhost:8082/redoc`

#### 4. 前端环境配置

```bash
cd ModelService_graduation-main/Front_UAV-main
npm install
npm run dev
```

前端默认运行在 `http://localhost:5174/` 或 `http://localhost:5176/`（由 Vite 自动选择）

## 📚 核心功能详解

### 1. 智程导航 - 智能路径规划

**功能描述**：
基于大语言模型（Qwen2.5）的自然语言路径规划系统，支持多策略路径规划与可视化。

**技术实现**：
- **自然语言解析**：使用LLM将自然语言指令（如"从A到B的最短路径"）转换为结构化参数
- **多策略规划**：最短路径、最快路径、避堵路径等
- **地图集成**：高德地图API集成，支持路径可视化
- **多目标优化**：通过加权综合评价函数选择最优路线

**API端点**：
- `POST /api/route/plan` - 创建路径规划
- `POST /api/chat` - 统一聊天入口（支持路径规划指令）

**前端页面**：
- `/route-planning` - 路径规划交互页面

---

### 2. 智眸千析 - 多模态图像识别

**功能描述**：
基于YOLO系列算法的多类目标检测与人物属性识别系统，结合LLM实现图像-文本多模态理解。

**技术实现**：
- **目标检测**：YOLOv5/YOLOv8进行人物检测、车牌检测等
- **人物属性识别**：年龄、性别、服装颜色等多维度特征识别
- **多模型融合**：YOLO + ResNet + 颜色识别的级联融合架构
- **多模态对话**：图像识别结果与LLM语义理解的深度融合，支持图像分析对话

**API端点**：
- `POST /api/image-recognition/analyze` - 图像识别与分析
- `POST /api/image-analysis-chat` - 图像分析对话

**前端页面**：
- `/image-recognition` - 图像识别交互页面

---

### 3. 智航监控 - 车牌监控与告警

**功能描述**：
车牌检测、识别与监控系统，支持视频分析与智能告警。

**技术实现**：
- **车牌检测**：YOLO检测车牌位置
- **车牌识别**：CNN+LSTM识别车牌字符，ColorNet识别车牌颜色
- **视频分析**：视频分帧处理、时序分析、关键帧提取
- **状态向量建模**：多源状态向量建模，支持精确匹配、子串匹配、相似度匹配
- **智能告警**：基于检测结果的智能决策与告警机制

**API端点**：
- `POST /api/plate-monitoring/upload-video` - 上传视频进行车牌监控
- `GET /api/plate-monitoring/task/{task_id}` - 获取监控任务结果

**前端页面**：
- `/plate-recognition` - 车牌监控页面
- `/data-dashboard` - 监控数据大屏

---

### 4. 邮件自动化 - 智能告警通知

**功能描述**：
基于检测结果的智能决策与邮件通知系统。

**技术实现**：
- **决策触发**：基于检测结果（目标车牌匹配、人物检测等）触发决策
- **告警摘要生成**：构造告警摘要，包含处理任务ID、时间戳、目标车牌号、匹配帧截图路径等
- **邮件通知**：通过邮件通知系统自动发送告警信息

**API端点**：
- 集成在车牌监控和图像识别模块中

**前端页面**：
- `/contact` - 联系页面（邮件发送）

## 🐛 常见问题

### Q1：端口被占用，后端启动失败？

```bash
# 检查端口占用
netstat -ano | findstr :8082

# 使用其他端口启动
python start_service.py --host 0.0.0.0 --port 8083
```

记得同步更新前端的代理配置。

### Q2：前端访问时报错：无法连接后端 / CORS 问题？

- 检查后端是否已成功启动、端口是否正确
- 确认前端 `vite.config.js` 中的代理地址与后端实际端口一致
- 检查 FastAPI 中的 CORS 中间件设置

### Q3：模型推理报错：找不到权重或 CUDA 错误？

- 确认权重文件是否已正确放置（部分大文件可能被 `.gitignore` 忽略）
- 检查 CUDA 版本与 PyTorch 等依赖是否匹配
- 若当前环境无 GPU，可尝试切换到 CPU 模式

### Q4：LLM API 调用失败？

- 检查 Qwen2.5 API 密钥配置是否正确
- 确认网络连接正常，API 服务可访问
- 查看后端日志，定位具体错误原因
