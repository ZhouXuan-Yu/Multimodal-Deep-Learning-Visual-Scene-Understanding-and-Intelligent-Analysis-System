一、项目功能结构拆解（聚焦四个核心子系统）

1.1 总体功能树状结构

- 前端交互层（Front_UAV-main）
  - 导航与任务规划子系统（智程导航）
    - 功能页：`PathPlanningPage.vue`（功能介绍页）
    - 应用页：`RoutePlanningView.vue`（交互式路径规划与可视化）
    - 头部导航：`TheHeader.vue` 中的「智程导航」入口
  - 智能人物识别子系统（智眸千析）
    - 功能页：`PersonRecognitionPage.vue`（功能与特性说明）
    - 应用页：`PersonRecognitionApp.vue`（图像上传、多特征识别与LLM交互）
    - 相关组件：`ImageUploader.vue`、`AnalysisResults.vue`、`AnalysisAssistant.vue`
  - 智航监控子系统（监控设备状态与数据大屏）
    - 数据大屏入口与布局：`DataDashboardView.vue`、`DataDashboardDetailView.vue`
    - 数据分析组件：`DataChartsComponent.vue`（多维度数据可视化）、`DataDrilldownComponent.vue`（数据钻取）、`DataComparisonComponent.vue`（数据对比）、`AnomalyDetectionComponent.vue`（异常检测）
    - 地理服务组件：`GeoApiDashboard.vue`（地理编码/逆地理编码、路径规划等）
  - 智能咨询与邮件子系统（自动发送邮件）
    - 联系页面：`ContactView.vue`
    - DeepSeek 调用逻辑：`processMessageWithDeepSeek`
    - EmailJS 邮件发送逻辑：`sendEmail`

- 后端服务层（ModelService/Main）
  - 导航与路径规划服务
    - 路线规划路由：`app/routers/route_planning.py`（/plan 与 create_route_plan、fallback_route_analysis）
    - 聊天统一入口：`app/routers/chat.py` 中 `handle_route_planning`
    - 第三方地图调用：`get_route_plan` → 高德地图 Driving API
  - 人物识别与特征分析服务
    - 图像识别主路由：`app/routers/image_recognition.py`
    - 深度学习模型封装：AgeModel、GenderModel、衣物颜色识别与 ImageProcessor
    - 视频级分析工具：`app/utils/video_analyzer.py`
    - 图像-LLM交互：`app/routers/image_analysis_chat.py`
    - mock 服务：`app/routers/mock_image_recognition.py`（用于前端联调与演示）
  - 统一聊天与任务编排服务
    - 文本/任务路由：`app/routers/chat.py`（根据 content.type 分发到路线规划、图像分析等子服务）

- 算法模型层
  - 路线规划算法
    - 基于 LLM 的语义解析与出行偏好抽取（Ollama + ROUTE_PROMPT）
    - 高德地图路径搜索与多策略（最短时间、最低费用、最短距离）规划
  - 人物特征识别算法
    - YOLO 系列模型用于人脸/人体检测与性别分类
    - ResNet50 等深度网络用于年龄估计与特征提取
    - 自定义衣物颜色分析与上、下装区域分割
    - 多模型结果融合与置信度计算（年龄、性别、颜色多维特征）
  - LLM 智能分析与文本生成
    - Ollama 本地大模型：路线规划意图解析与参数结构化
    - DeepSeek Chat API：用户需求理解与咨询答复（ContactView）

- 数据层与第三方服务
  - 本地数据库：`db.sqlite3`（任务、日志、配置等）
  - 临时数据与中间结果：`temp/`、`uploads/`、`output/`
  - 高德地图 REST API（地理编码、逆地理编码、路径规划、区域查询等）
  - EmailJS 邮件服务（邮件模板、服务 ID、公钥配置）

- 系统集成层
  - 前后端接口规范：
    - 路径规划：前端 `RoutePlanningView.vue` 通过 `useMainStore().sendChatMessage` 向后端 `/chat/completions` 或 `/plan` 发起请求
    - 人物识别：前端 `PersonRecognitionApp.vue` 通过 `personRecognitionService.analyzeImage` 上传图片至 `/image-recognition` 或 `/mock_image_recognition/analyze`
    - 邮件发送：`ContactView.vue` 直接调用 DeepSeek HTTP API 与 EmailJS 前端 SDK
  - 跨模块协同：
    - 路线规划结果以结构化 JSON 返回，前端映射为推荐路线列表、地图覆盖物与侧边栏导航步骤
    - 人物识别结果统一为 `persons` 列表（年龄、性别、颜色、bbox 等字段），前端进行可视化标注与自然语言问答
    - 智航监控数据大屏将多维度分析、异常检测与地理信息服务进行统一编排展示
    - 联系表单将用户业务需求映射为 LLM 友好的 Prompt，再转化为结构化邮件内容发送给运维/业务方

1.2 智程导航模块分析

- 核心目标：构建一个支持自然语言交互、融合高德地图与本地 LLM 的智能路径规划系统，为车辆等任务规划提供可视化导航方案。
- 输入：
  - 用户自然语言描述（前端聊天窗口/快捷按钮预置 Prompt）
  - 历史路线上下文（向量检索得到的历史任务）
  - 地理位置与城市编码信息（前端城市数据、用户起终点选择）
- 输出：
  - 推荐路线集合（多策略路径、推荐/备选标签、推荐理由）
  - 路径详细信息（起点、终点、途经点、行驶时间、距离、过路费、限行信息、途经城市）
  - 可视化元素（地图折线、路况图层、卫星图层、3D 建筑）
- 关键算法或技术：
  - LLM Prompt 设计与结构化结果解析（ROUTE_PROMPT + JSON 强约束）
  - 基于高德 Driving API 的多策略路径搜索与路况融合
  - 历史任务向量检索与上下文增强（Vector Store）
  - 前端交互式路线推荐与多维指标对比（时间/距离/费用）
- 数据流向：
  - 前端：用户在 `RoutePlanningView.vue` 输入自然语言 → 通过 store 封装为请求参数 → 调用后端 `/chat/completions` 或 `/plan`
  - 后端：`chat.py` 中 `handle_route_planning` 或 `route_planning.py` 解析请求 → 调用 Ollama LLM 获取结构化路线意图 → 调用高德 API 获取真实路径细节 → 整合为 `RouteResponse` 返回
  - 前端：解析 `RouteResponse` → 更新推荐路线列表、地图渲染状态及路线详情面板
- 模块层级归属：
  - 前端交互层：`PathPlanningPage.vue`、`RoutePlanningView.vue`、地图与引导组件
  - 后端服务层：`route_planning.py`、`chat.py`
  - 算法模型层：Ollama 大模型、路线规划 Prompt 策略
  - 数据层：历史路线向量库、高德地图 API 返回数据
  - 系统集成层：前后端接口与第三方 API 协同

1.3 智眸千析模块分析

- 核心目标：构建一个面向复杂监控场景的人物多特征识别与交互分析系统，实现性别、年龄、服装颜色等属性的联合识别，并支持基于自然语言的多模态检索与分析。
- 输入：
  - 用户上传的单帧图片或视频帧
  - 可选模式参数（普通识别/增强识别等）
  - 用户针对识别结果的自然语言提问（通过前端对话组件）
- 输出：
  - 人物检测结果列表（每个人物的 bbox、年龄、性别、上衣颜色、下衣颜色及置信度）
  - 关联的可视化信息（人物框选、高亮显示、属性标签）
  - 基于识别结果的文本分析与解释（LLM 对识别结果进行总结与语义回答）
- 关键算法或技术：
  - YOLO 系列模型进行人脸/人体检测与性别分类
  - ResNet50 及改进网络完成年龄估计与高维特征提取
  - 自定义衣物区域分割与颜色分析算法
  - 多模型结果融合策略（基于置信度的加权融合与边界框一致性约束）
  - 图像特征与 LLM 的多模态对齐与问答接口（image_analysis_chat）
- 数据流向：
  - 前端：`PersonRecognitionApp.vue` 接收用户上传图片 → 构造 FormData → 调用 `personRecognitionService.analyzeImage` → 获得后端返回的 `persons` 结构
  - 后端：`image_recognition.py` / `mock_image_recognition.py` 读取图像 → 通过 YOLO/ResNet 等模型完成检测与特征分析 → 将结果封装为统一 JSON（年龄、性别、颜色、bbox、置信度等）→ 返回前端
  - 若启用图像问答：`image_analysis_chat.py` 将识别结果与用户问题联合输入 LLM，生成自然语言解释
- 模块层级归属：
  - 前端交互层：`PersonRecognitionPage.vue`、`PersonRecognitionApp.vue` 及人物分析相关组件
  - 后端服务层：`image_recognition.py`、`mock_image_recognition.py`、`image_analysis_chat.py`
  - 算法模型层：AgeModel、GenderModel、衣物颜色识别模型、VideoAnalyzer 等
  - 数据层：上传图片文件、临时结果图像、识别日志
  - 系统集成层：图像上传接口、识别服务与 LLM 分析服务的集成

1.4 自动发送邮件功能模块分析

- 核心目标：在联系页面中引入 LLM 驱动的智能咨询摘要与邮件自动发送机制，实现用户业务需求的结构化归纳与自动分发，提高项目沟通效率。
- 输入：
  - 用户基本信息（姓名、邮箱、电话、公司、微信号）
  - 用户感兴趣的功能类型（路径规划、人员识别、车辆监控、灾害检测等）
  - 用户自由文本留言（需求描述、场景说明、合作意向等）
- 输出：
  - 由 DeepSeek 生成的结构化咨询摘要（标题、大纲、内容、建议、解决方案）
  - 经过 HTML 模板渲染的邮件内容（包含客户信息卡片、AI 分析结果与原始留言）
  - 发送至指定接收邮箱的咨询邮件（EmailJS 服务）
- 关键算法或技术：
  - Prompt 工程：在 `processMessageWithDeepSeek` 中融入项目总体功能说明，引导模型从项目能力视角为用户需求匹配合适方案
  - 结构化文本解析：`formatProcessedContent` 通过正则与多策略解析，将 LLM 输出拆分为标题、大纲、建议等子字段
  - HTML 模板生成：在前端构建层次清晰、信息分区明确的邮件 HTML，以便人工快速理解用户需求
  - 失败回退策略：当 DeepSeek 调用失败或结构不符合预期时，使用固定模板 `formatFallbackResponse` 自动生成合理的回复结构
  - EmailJS 集成：通过前端 SDK 调用 EmailJS 的服务 ID 与模板 ID，实现邮件发送与模板变量渲染
- 数据流向：
  - 前端：`ContactView.vue` 收集表单数据 → 构造包含项目功能说明与用户描述的 Prompt → 调用 DeepSeek API 获取结构化分析 → 解析为 `formattedContent` → 与用户信息一起渲染 HTML → 调用 EmailJS 发送邮件
  - 第三方服务：DeepSeek Chat 负责自然语言理解与结构化生成，EmailJS 负责邮件投递
- 模块层级归属：
  - 前端交互层：`ContactView.vue`
  - 后端服务层：无直接后端参与（全部在前端 + 第三方服务完成）
  - 算法模型层：DeepSeek LLM、Prompt 模板与解析逻辑
  - 数据层：用户表单数据、AI 生成咨询摘要、邮件发送日志（由 EmailJS 维护）
  - 系统集成层：前端与 DeepSeek、EmailJS 的直接集成

1.5 智航监控模块分析

- 核心目标：构建面向监控数据的综合分析平台，实现多维度数据分析、可视化展示与地理信息服务的统一管理，为运营决策提供数据支撑。
- 输入：
  - 监控设备运行状态数据（电量、信号强度等）
  - 多维度监控数据（人员监测、车辆监测、风险事件等）
  - 地理信息查询需求（地理编码、逆地理编码、路径规划、区域查询等）
- 输出：
  - 多维度数据分析面板（人物识别分布、活动趋势、任务执行情况、风险分析等）
  - 设备状态监测图表（电量趋势、信号强度、飞行速度等）
  - 数据钻取与对比分析视图
  - 异常检测与预警信息
  - 地理信息服务结果（地理位置查询、路径规划、周边设施检索等）
- 关键算法或技术：
  - 多维数据可视化：基于 ECharts 实现时序数据（电量、信号、速度）、饼图（人物识别分布）、堆叠柱状图（风险分析）等多种图表类型的统一渲染与管理
  - 数据钻取与对比分析：通过交互式筛选与多维度聚合，支持从宏观数据逐层深入到细粒度数据，支持时间维度与区域维度的对比分析
  - 异常检测算法：基于统计分析方法，对设备运行数据进行实时监测，自动识别异常波动并触发预警
  - 地理信息服务集成：集成高德地图地理编码/逆地理编码、路径规划、区域查询等 API，为监控数据提供空间维度支撑
- 数据流向：
  - 前端：`DataDashboardDetailView.vue` 聚合 `DataChartsComponent.vue`（多维度数据图表）、`DataDrilldownComponent.vue`（数据钻取）、`DataComparisonComponent.vue`（数据对比）、`AnomalyDetectionComponent.vue`（异常检测）与 `GeoApiDashboard.vue`（地理服务），构成统一的数据分析平台
  - 第三方服务：高德地图 REST API 提供地理编码、逆地理编码、路径规划、区域查询等空间信息服务
- 模块层级归属：
  - 前端交互层：`DataDashboardView.vue`、`DataDashboardDetailView.vue` 及数据分析/地理服务相关组件
  - 前端组件层：`DataChartsComponent.vue`、`DataDrilldownComponent.vue`、`DataComparisonComponent.vue`、`AnomalyDetectionComponent.vue`、`GeoApiDashboard.vue`
  - 第三方服务层：高德地图 REST API
  - 系统集成层：前端组件与第三方 API 的直接集成

二、论文可写的核心技术点提炼（围绕四个功能模块）

2.1 智程导航可写技术点

- 可上升为论文技术点的内容：
  - LLM 驱动的多策略路径规划框架：将用户自然语言需求解析为结构化路线参数，并结合高德地图实现多目标路径优化。
  - 历史路线向量检索与个性化推荐机制：利用向量数据库检索相似历史任务，为当前路径规划提供上下文增强。
  - 前后端协同的多维指标可视化：将时间、距离、费用、限行等信息统一映射为可交互仪表盘与地图叠加层。
- 建议的小节标题（示例）：
  - "基于大模型语义解析的多策略路径规划方法设计"
  - "融合历史任务知识的个性化路线推荐机制"
  - "面向多设备监控任务的路径规划可视化交互设计"
- 可展开的技术论述方向：
  - 不同 Prompt 模板与温度参数对路线规划稳定性的影响分析
  - 高德多策略路径与用户出行偏好（避拥堵、避收费）之间的协同优化
  - 前端可视化交互对用户决策效率与任务规划准确性的影响
- 是否可以上升为优化设计或创新点：
  - 若进一步引入统一的多目标优化（时间/费用/安全性）与在线学习机制，可作为"方法创新点"。
  - 当前 LLM + 高德的组合方式，可作为"系统架构设计与人机交互优化"的工程创新点。

2.2 智眸千析可写技术点

- 可上升为论文技术点的内容：

  - 基于场景的衣物颜色识别与区域分割算法：利用人脸位置推断身体区域，提升服装特征识别准确率。
  - 图像识别结果与 LLM 的多模态融合：将结构化人物特征转化为可对话语境，实现"按描述检索人物"的交互。
- 建议的小节标题（示例）：
  - "面向复杂监控场景的多特征人物识别模型设计"
  - "基于服饰区域分割的颜色识别与特征表达方法"
  - "人物特征与自然语言的多模态关联与交互机制"
- 可展开的技术论述方向：
  - YOLO、ResNet 等不同模型在本项目场景下的精度与推理速度对比
  - 不同服饰区域划分策略对颜色识别鲁棒性的影响
  - 将识别结果嵌入 LLM Prompt 后，对回答质量和解释性带来的提升
- 是否可以上升为优化设计或创新点：
  - 若在服饰颜色识别或多特征融合策略上给出明确的算法设计与实验对比，可作为算法创新点。
  - 多模态问答与监控场景结合的应用，可以作为"应用创新 + 工程实现创新"的综合亮点。

  - 数据钻取与多维对比分析机制：支持从宏观概览到细粒度数据的逐层深入，提供时间维度与空间维度的对比分析能力。
  - 设备运行异常检测算法：基于统计分析的实时异常识别与预警机制，实现设备状态的主动式健康监测。
  - 地理信息服务集成与空间数据增强：集成高德地图地理编码/逆地理编码、路径规划等 API，为监控数据提供空间维度支撑。

四、创新点挖掘（面向四个功能的综合分析）

4.1 方法创新
- 将 LLM 语义解析与传统地图路径规划算法结合起来，实现从自然语言到多策略路线参数的自动映射。
- 通过多模型融合结构，将人物的年龄、性别、服装颜色等多维特征统一到单一接口，便于上层多模态交互。
- 在工程沟通场景中引入 LLM 自动摘要与模板化邮件生成，构建"智能咨询前置层"。
- 在监控场景中设计多维度数据可视化框架，实现时序数据与分布数据的统一渲染与管理。

4.2 应用创新
- 将路径规划、人物识别、智航监控与智能咨询整合到同一多设备监控应用平台中，面向实际安防、巡检等场景提供一体化方案。
- 智眸千析模块支持"按自然语言描述检索人物"，适用于复杂监控与安防排查场景。
- 智航监控模块通过数据钻取与多维对比分析，支持从宏观到微观的逐层数据探索。
- 自动邮件模块可扩展为智能工单/客服系统，适应高校科研项目、企业方案定制等应用。

4.3 系统架构创新
- 采用"前端富交互 + 轻量后端服务 + 本地 LLM + 外部 API"的混合式架构，兼顾可扩展性与部署灵活性。
- 后端通过统一聊天路由与任务派发机制，将路线规划、图像识别等功能统一到一个对话式接口之下。
- 前端通过组件化设计，实现数据分析组件与地理服务组件的灵活组合与复用。

4.4 多模型融合创新
- 在人物识别场景中，融合 YOLO、ResNet 与颜色分析模块，实现多维特征协同识别。
- 在导航场景中，将 LLM 规划结果与高德实际道路数据进行二次融合，既保证语义合理性又保证可执行性。
- 在智航监控场景中，融合 ECharts 可视化引擎与高德地图 API，实现数据与空间的一体化展示。

4.5 工程实现创新
- 通过 mock 服务与统一接口设计，降低前后端联调耦合度，提升开发与演示效率。
- 在前端直接集成 DeepSeek 与 EmailJS，简化后端负载，将部分智能处理逻辑前移到浏览器侧。
- 通过数据钻取与多维对比分析组件的设计，实现数据分析功能的模块化与可配置化。
