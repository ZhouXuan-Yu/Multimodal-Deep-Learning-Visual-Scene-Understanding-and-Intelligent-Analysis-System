# ModelService 项目启动指南

## 📋 项目概述

本项目包含前后端分离的AI模型服务系统：
- **后端**: FastAPI + AI模型服务 (端口: 8081)
- **前端**: Vue.js + Vite (端口: 5176)

## 🚀 快速启动

### 环境要求
- Python 3.8+
- Node.js 16+
- CUDA 12.4+ (GPU支持)
- conda环境: `modelapp`

### 1. 后端启动

#### 方法一: 使用统一启动脚本 (推荐)
```bash
# 进入项目根目录
cd D:\ModelService_graduation-main\ModelService_graduation-main

# 激活conda环境
conda activate modelapp

# 启动后端服务 (注意: 如果8081端口被占用，使用8082)
python start_service.py --host 0.0.0.0 --port 8082
```

#### 方法二: 直接启动FastAPI
```bash
# 进入后端目录
cd D:\ModelService_graduation-main\ModelService_graduation-main\ModelService\Main

# 激活conda环境
conda activate modelapp

# 启动服务
python -m uvicorn app.main:app --host 0.0.0.0 --port 8081 --reload
```

### 2. 前端启动

#### 安装依赖 (首次运行)
```bash
# 进入前端目录
cd D:\ModelService_graduation-main\ModelService_graduation-main\Front_UAV-main

# 安装依赖
npm install
```

#### 启动前端服务
```bash
# 启动开发服务器
npm run dev
```

### 3. 访问地址

启动完成后，访问以下地址：

- **前端界面**: http://localhost:5174/ (Vue.js + Vite)
- **后端API文档**: http://localhost:8082/docs (FastAPI自动生成)
- **Swagger UI**: http://localhost:8082/docs
- **ReDoc文档**: http://localhost:8082/redoc

**注意**: 前端端口可能因系统占用而变化，通常为5174-5179之间的某个端口。

前端开发服务器已配置代理，将API请求转发到后端：
```javascript

  '/api': {
    target: 'http://localhost:8081',
    changeOrigin: true
  }
```

## ⚠️ 常见问题

### 1. 后端启动失败

**问题**: 找不到车牌识别服务
```bash
ERROR - 车牌识别服务脚本不存在: D:\ModelService_graduation-main\ModelService_graduation-main\design-competition\appPro.py
```

**解决**: 项目已修复路径问题，请使用上述启动命令。

### 2. 前端缺少依赖

**问题**: 找不到视频文件
```bash
Failed to resolve import "../../assets/videos/file.mp4"
```

**解决**: 项目已处理缺失视频文件问题，可以正常启动。

### 6. 前端API请求失败

**问题**: 前端仍然连接旧端口(8081)
```bash
POST http://localhost:8081/api/night-detection/process net::ERR_CONNECTION_REFUSED
```

**解决**:
1. **修改主配置文件**: `src/config.js` (默认端口)
2. **修改domain API配置**: `src/views/domain/api/config.js`
3. **修改port配置**: `src/views/domain/src/port_config.js` (BACKEND_PORT)
4. **修改服务文件**: `src/views/domain/services/nightEnhancementService.js`
5. **重启前端服务**: `npm run dev`

**注意**: 前端有多层配置文件和不同的request模块，需要逐一检查和更新。

### 调试信息
前端现在包含详细的调试日志：
- `[前端请求]` - 显示基础URL和请求URL
- `[domain/src/utils/request]` - domain模块的详细请求信息
- `[前端接收]` - 显示完整的响应信息

这些日志将帮助快速定位API连接问题。

### 7. 夜间检测模块无法加载

**问题**: night_detection模块被标记为"不可用"
```bash
模块不可用，跳过路由注册: night_detection
```

**根本原因**: Pillow库的_imaging.dll加载失败
```bash
DLL load failed while importing _imaging: 系统无法执行 %1
```

**解决**:
1. **重新安装Pillow**:
   ```bash
   pip uninstall pillow -y
   pip install pillow
   ```
2. **验证安装**: 确保PIL可以正常导入和使用
3. **重启服务**: 重新启动后端服务以加载修复的模块

**注意**: 这通常是由于Pillow版本兼容性或系统依赖问题导致的。

### 3. 模块导入错误

**问题**: No module named 'app.route_config'
**解决**: 已修复Python路径问题。

### 4. GPU支持

**问题**: CUDA capability警告
**解决**: 已安装CUDA 12.4版本PyTorch，支持RTX 5060。

### 5. 端口占用错误

**问题**: 服务启动后立即退出，显示端口占用错误
```bash
ERROR: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8081): 通常每个套接字地址(协议/网络地址/端口)只能使用一次
```

**解决**:
1. **使用不同端口启动**:
   ```bash
   python start_service.py --host 0.0.0.0 --port 8082
   ```

2. **终止占用端口的进程**:
   ```bash
   # 查看占用端口的进程
   netstat -ano | findstr :8081

   # 终止进程 (将<PID>替换为实际进程ID)
   taskkill /F /PID <PID>
   ```

3. **清理系统端口**:
   ```bash
   # 等待系统释放端口 (通常需要30秒-1分钟)
   timeout /t 30
   ```
