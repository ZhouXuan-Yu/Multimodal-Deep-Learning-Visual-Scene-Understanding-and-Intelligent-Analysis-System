## 🔧 配置管理

### 后端配置位置

#### 1. 路由配置 (`app/route_config.py`)
```python
API_ROUTES = {
    "night_detection": {
        "prefix": "/api/night-detection",
        "tags": ["低光图像增强与目标检测"],
        "description": "夜间低光图像增强与目标检测",
        "enabled": True,
        "frontend_path": "/night-detection"
    },
    # ... 其他路由配置
}
```

#### 2. 路由注册 (`app/main.py`)
```python
# 动态注册路由
for module_name, module in modules.items():
    if hasattr(module, 'router'):
        prefix = getattr(module, 'API_PREFIX', f'/api/{module_name.replace("_", "-")}')
        app.include_router(module.router, prefix=prefix)
```

### 前端配置位置

#### 1. 主配置 (`src/config.js`)
```javascript
const API_BASE_URL = 'http://localhost:8082/api';
```

#### 2. Domain API配置 (`src/views/domain/api/config.js`)
```javascript
export const API_BASE_URL = 'http://localhost:8082/api';
```

#### 3. Port配置 (`src/views/domain/src/port_config.js`)
```javascript
export const BACKEND_PORT = 8082;
```

#### 4. Vite代理配置 (`vite.config.js`)
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8082',
    changeOrigin: true
  }
}
```

## 📝 API端点详情

### 夜间检测模块 (`/api/night-detection`)

| 端点 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/process` | POST | `image: UploadFile` | 处理单张图像 |
| `/process-video` | POST | `video: UploadFile, night_vision_mode: bool, enhance_method: str, conf: float` | 处理视频 |
| `/video-progress/{process_id}` | GET | `process_id: str` | 获取视频处理进度 |

### 车牌识别模块 (`/api/plate-recognition`)

| 端点 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/process` | POST | `image: UploadFile` | 处理车牌图像 |
| `/batch-process` | POST | `images: List[UploadFile]` | 批量处理车牌图像 |

### 路径规划模块 (`/api/route`)

| 端点 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/plan` | POST | `start: dict, end: dict, waypoints: list` | 规划路径 |
| `/optimize` | POST | `route: dict, constraints: dict` | 优化路径 |

## ⚠️ 已知问题

### 1. 夜间检测模型缺失
**问题**: 无法加载必要的模型文件
**原因**: `night recognition/Night/new_detection/main/model.py` 文件不存在
**解决方案**: 需要恢复或重新创建模型文件

### 2. 依赖缺失
**问题**: 某些模块因缺少依赖包而不可用
- `fire-detection`: 缺少 `sqlalchemy`
- `video-tracking`: 缺少 `mediapipe`
- 其他模块: 缺少相应依赖

### 3. 前端配置不一致
**问题**: 前端有多个API配置位置，需要保持同步
**解决方案**: 修改时需要同时更新所有相关文件

## 🔄 路由修改流程

### 添加新路由

1. **后端**:
   - 在 `app/route_config.py` 中添加配置
   - 创建路由模块文件 `app/routers/{module_name}.py`
   - 确保模块能正确导入

2. **前端**:
   - 更新 `src/config.js`
   - 更新 `src/views/domain/api/config.js`
   - 更新 `src/views/domain/src/port_config.js`
   - 更新 `vite.config.js` 中的代理配置

### 修改现有路由

1. 确保前后端路径保持一致
2. 更新所有相关配置文件
3. 测试API调用是否正常
